Philosophy

  • Single source of truth: "some_document_v2_final_actual_final_version(3).xlsx" makes me cry inside
  • Work smarter, not harder: Can this be done in a better/quicker way? Can I create a Python script for this?
  • Automation: Automate as much as possible
  • Incremental Improvement: Frequent small changes; small/simple PRs
  • Scalability/Robustness: If we're going to build it, build it well/future proof
  • Transparency/Openness/Sharing: Sharing is caring; no reinventing the wheel; everyone knows what others are working on
  • Document, Document, Document: Limit sources of failure
  • Collaboration: More brains = more ideas/viewpoints = better solutions/outcomes
  • Cloud/Live Collaboration: No saving of documents on personal devices; autosave!; no attaching of files to emails (links only)

Project Management

Despite no formal PM training, during my time as a Data Lead/Project Manager, I have developed the following:

  • Requirements Documents: To outline Objectives, Background, User Stories, Design, Deliverables, Task List
  • Reflection Documents: To reflect on things that went well, and things that could be improved upon next time; making mistakes = learning
  • Information Packs: To provide critical information to stakeholders for complex deliverables
  • Review Meetings: To hear feedback from stakeholders
  • Change Request/Improvement Lists: List of requested/identified improvements to deliverables
  • Issue Resolution Documents: List any issues/blockers, their impacts, and brainstorm possible solutions
  • Report Register: List of available dashboards and information

What I work by

  • Coding
    • Keep code DRY
    • Use of parameterisation and reusable functions/modularity
  • Spreadsheets
    • I believe spreadsheets should be a project management tool only. In a modern data world, they should not be used for non ad hoc data analysis and graphs!
  • Dashboarding
    • I am capable of developing in Power BI, however I appreciate freedom to use code-based dashboarding tools (eg. Streamlit) for improved:
      • Efficiency: Reduction of manual and time consuming clicking/dragging
      • Flexibility: Use of Python packages, customisable UI using HTML/CSS, and greater selection and customisation of visuals
      • Increased control/reduction of errors: The logic is readable via code, not buried within menus/clicks
      • Version control: Simply rollback rather than contend with a fire ("oops I broke the report, now I have to figure out a fix right now")
